# AI Assistant Project Guide

This document outlines the current capabilities and future development roadmap for the Adaptive AI assistant within AdaptiveCRM.

## Current AI Capabilities (Completed)

The AI assistant can currently understand and perform the following actions based on natural language commands.

- [x] **Answer Questions**: Respond to queries about CRM data (e.g., "Who are my contacts at Innovate Inc.?", "Show me my high-priority tasks").
- [x] **Create Task**: Create a new task and associate it with a contact.
  - *Example*: "Create a task to follow up with Alana Patel by next Friday."
- [x] **Create Opportunity**: Create a new sales opportunity.
  - *Example*: "New opportunity for a website redesign with Innovate Inc., value is 25000, close date is end of August."
- [x] **Log Activity**: Log activities like calls, emails, meetings, and notes.
  - *Example*: "Log a 15-minute call with Ben Carter about cloud services."
- [x] **Create Lead**: Create a new lead.
  - *Example*: "Create a new lead for Jane Smith from Tech Innovators, her email is jane.s@techinnovators.com."
- [x] **Update Task**: Modify an existing task (e.g., change due date, status, priority).
  - *Example*: "Change the priority of the proposal follow-up task to High."

### IMPORTANT: Application Flow

The application should follow this general data and user flow:

1.  **App Load**: `index.html` loads `index.tsx`, which mounts the main `App.tsx` component.
2.  **Auth Check**: A top-level component or hook (e.g., `useAuth`) checks for an active user session with Supabase Auth.
    - If no user, redirect to a new `/login` page.
    - If user exists, proceed.
3.  **Data Fetching**: After successful authentication, fetch all necessary CRM data for the authenticated user/company from Supabase tables (e.g., contacts, tasks).
4.  **Render UI**: Display the application UI with the fetched data. Use a loading state during the fetch.
5.  **User Interaction**: A user performs an action (e.g., clicks "Add Task").
6.  **Backend Call**: The action triggers a Supabase function (e.g., `supabase.from('tasks').insert(...)`).
7.  **State Update**: The UI updates to reflect the change. This should be handled by a global state management solution and can be enhanced with Supabase Realtime for instant updates across all clients.

### IMPORTANT: AI Setup Details

The AI Assistant's logic is primarily in `AIAssistantModal.tsx`.

-   **System Prompt**: A detailed system instruction defines the AI's role, its available tools (actions like `create_task`), and the required JSON format for function calls. It's crucial for controlling the AI's behavior.
-   **Data Context Injection**: On each new user query, a snapshot of the current CRM data (contacts, tasks, etc., from `appContext`) is stringified and appended to the user's prompt. This makes the AI "context-aware" for that specific turn, allowing it to answer questions like "Who are my contacts?" without needing a long-term memory.
-   **Action Execution Flow**:
    1.  The AI receives the user query + data context.
    2.  It determines if an action is needed and if it has all the required parameters.
    3.  If information is missing, it asks a clarifying question.
    4.  If it has all information, it responds with a message and a ````json ... ```` block containing the action name and parameters.
    5.  The frontend parses this JSON block.
    6.  The `handleExecuteAction` function in the modal calls the appropriate `appContext.set...` function.
    7.  **CRITICAL REFACTOR**: All calls within `handleExecuteAction` must be changed from local state setters (`appContext.setTasks`) to direct Supabase client calls (`supabase.from('tasks').insert(...)`).

### IMPORTANT: SQL Database Schema

SQL schemas and seed data for each table are documented in individual Markdown files within the `sqlTables/` directory. Each file contains a description and the necessary SQL commands (including `CREATE TYPE` and `CREATE TABLE`) to set up and populate a table. These can be run in the Supabase SQL Editor to set up the database for development.

# Tasks Table Schema

The `tasks` table is used to store both project and case tasks in a unified structure. This reduces duplication and simplifies querying across the application.

### IMPORTANT: Environment Variables & System Settings

A `.env.local` file (or platform-equivalent secrets management) is required with the following variables:

-   `API_KEY`: The Google Gemini API key for all AI features.
-   `SUPABASE_URL`: The URL for your Supabase project.
-   `SUPABASE_ANON_KEY`: The anonymous public key for your Supabase project.

**System Defaults & Placeholders to Replace:**

-   **Image Placeholders**: The app currently uses `https://picsum.photos` and `https://i.pravatar.cc` for mock images. All image URLs (`avatarUrl`, `imageUrl`, etc.) must be replaced with URLs from **Supabase Storage**.
-   **Default User**: The `currentUser` is hardcoded as the team member with `id: 'user'`. This must be replaced with the dynamically fetched, authenticated user from `supabase.auth.user()`.

#### a. Client & Database Setup
- [x] **Client**: Create a `lib/supabaseClient.ts` to initialize and export the Supabase client.
- [x] **Environment**: Create a `.env.local` file for environment variables (`SUPABASE_URL`, `SUPABASE_ANON_KEY`, `API_KEY`).
- [ ] **Schema**: Finalize the database schema based on `sqlTables/` and set up tables, relationships, and RLS (Row Level Security) policies in the Supabase SQL Editor.

#### b. Authentication
- [x] **Auth UI**: Create Login, Sign Up, and Forgot Password pages/components. (Login page implemented)
- [x] **`useAuth` Hook**: Implement a custom hook (`useAuth`) that manages user session, profile data, and auth state.
- [x] **Protected Routes**: Wrap the main application in a higher-order component that checks for an authenticated user and redirects to login if necessary.

#### c. Database CRUD & Realtime
- [ ] **CRUD Operations**: Replace all `set...` state functions with Supabase CRUD functions as mapped out in the **CRUD Endpoint Implementation Plan** below.
- [ ] **Realtime Subscriptions**: Implement Supabase Realtime subscriptions to listen for database changes (especially for `Chat.tsx` and `Activities.tsx`) and update the UI automatically.

#### d. Storage
- [ ] **File Uploads**: Re-wire `UploadFileModal` to upload files to a Supabase Storage bucket.
- [ ] **File Access**: Update all components that display images (`avatarUrl`, `logoUrl`, etc.) to use signed URLs from Supabase Storage.

#### e. Vector (for advanced AI)
- [ ] **Embeddings Function**: Create a Supabase Edge Function to generate embeddings from text data (e.g., task descriptions, case notes) and store them in a `vector` column.
- [ ] **Semantic Search**: Implement a search feature that uses the vector store to find semantically similar records.

#### f. Edge Functions (for secure backend logic)
- [ ] **Stripe Webhooks**: Create an Edge Function to securely handle webhooks from Stripe for billing.
- [ ] **Secure Emailing**: Create an Edge Function to handle sending emails, keeping the provider's API key secure.
- [ ] **Secure AI Calls**: Move AI-powered logic (like `generateAIInsights` and document generation) into Edge Functions to protect the Gemini API key.

### 2. State Management Refactor
- [ ] **Initial Fetch**: Replace initial data loading from `data.ts` with a fetch from Supabase tables on app load.
- [ ] **Global State**: Transition from passing state and setters as props (`appContext`) to a more robust global state solution (e.g., Context API with `useReducer`, Zustand, or Redux Toolkit).

### 3. Incomplete Component Logic
- [ ] **Modals with Actions**:
  - `ConfirmDeleteModal`: The `onConfirm` prop must call a Supabase delete function.
  - `InviteMemberModal`: The `onInvite` function needs to trigger `supabase.auth.admin.inviteUserByEmail()`.
- [ ] **Component Functions**:
  - `handleCreateInvoices` in `TimeBilling.tsx`: Must create invoices in the Supabase `invoices` table.
  - `handleConvertLead` in `Leads.tsx`: Must perform `insert` and `update` operations on the `leads`, `contacts`, and `opportunities` tables.
  - `handleStripeConnect` in `company/BillingIntegration.tsx`: Needs to initiate the OAuth flow with Stripe.
  - `handleResolveTicket` in `Chat.tsx`: Must update the status of a ticket in the `support_tickets` table.
  - `handleSaveGeneratedDocument` in `CaseDetailsModal.tsx`: Must upload the generated text as a file to Supabase Storage.
- [ ] **AI Refactor**:
  - `generateAIInsights` in `Dashboard.tsx`: This function should be moved to a Supabase Edge Function to protect the API key. The frontend should call this function instead of the Gemini API directly.
  - `AIAssistantModal`: All Gemini API calls within this component must be moved to a secure Supabase Edge Function.
- [ ] **Public Pages**:
  - `PublicEstimatePage`: The `onConvert` function needs to update the status in the `estimates` table.
  - `PublicInvoicePage`: The "Pay Now" button needs to integrate with Stripe to process payments.
